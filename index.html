泛型总结

Posted on 2018-07-21

—泛型:
在JDK1.4版本之前,容器什么类型都可以进行存储，但是在取出时，需要用到对象的特有内容时，需要向下转型
但是对象的类型不一致，导致了向下转型发生了类型转换异常。
为了避免这个问题，只能在主观上进行控制，往集合中存储对象时候必须保持一致
jdk1.5以后，解决了该问题，在定义集合时候，就直接明确集合中存储元素的具体类型，
这样，编译器在编译时候，就可以对集合中存储的对象类型进行检查
一旦发现了类型不匹配，就编译失败，这个技术就是泛型技术
好处：
1将运行时候的问题转移到了编译时期，可以更好的让程序员发现问题并解决问题
2避免了向下转型的麻烦
总结：
泛型就是应用在编译时期的一项安全机制
泛型的擦除：
编译器通过泛型对元素类型进行检查，只要检查通过，就会生成class文件，但是在class文件中就将泛型标识去除了
泛型的表现：
泛型技术在集合框架中，应用范围很大。
什么时候写泛型？
只要看到类或者接口在描述时右边有<>，就需要使用泛型，其实是，容器在不明确操作元素的前提下，对外提供了一个参数<>
使用容器时，具体的类型实参传递给该参数即可
说白了，泛型技术就是传递类型参数

集合整理

Posted on 2018-07-19

Collection接口中的共性方法
1添加：
boolean add(Object obj)
boolean addAll(Object obj)
2删除
void clear();所有
boolean remove(Object obj)一个
boolean removeAll(Collection c)一部分——retainAll()保留相同的
获取长度
int size();
判断
boolean isEmpty();
boolean Contains(Object obj);
boolean ContainsAll(Collection c);
集合转换为数组
toArrays();
取出集合的元素
Iterator iterator();
获取集合中元素上迭代功能的迭代器对象。
迭代：取出对象的一种方式
迭代器：具备迭代功能的对象
而迭代器对象不需要new,直接通过iterator()方法获取即可——通过内部类的方式
迭代器是取出Collection集合元素的公共方式
Collection接口的分类：
set：不允许有重复元素
list：有序（存入的顺序和取出的元素顺序一致，有索引，允许重复元素）
重点：List接口中的特有方法：它的特有方法都是围绕索引进行的
支持增删改查
1增
add(index,element)
2删
remove(index);
3改
set(index,newelement)
4查
int indexOf(element);
element get(index);

list集合的具体子类，子类之所以进行区分是因为内部的数据结构(内部的存储方式)不同。
–Vector:数据结构是数组，数组长度是可变的（不断new新数组并将原数组元素复制到新数组——最早出现的集合对象1.1——线程同——加锁变慢，故被淘汰_查和增删都慢
–ArrayList:也是数组结构，也是长度可变的。1.2——线程不同步的，替代了Vector——增删慢，查询快，
–LinkedList:链表结构，线程不同步——增删快，查找慢
set集合：不允许重复元素。和Collection的方法相同，set集合取出来的方法只有一个：迭代器
–HashSet:不保证顺序，可以有序，也可以无序。哈希表（散列表）结构——不同步
哈希：不允许存储重复元素，因为会发生查找的不确定性。
不保证存入和取出的顺序一致
比数组的查询效率高
用于存储元素和哈希值对应关系的容器称之为哈希表
当哈希算法算出的2个元素值相同，称之为哈希冲突。冲突后，进行进一步的判断，判断的是元素的内容，equals方法解决冲突。堆使用哈希，所以每个对象都有hashCode（）方法
哈希表在判断元素是否相同，依据hashCode().如果哈希值相同，在判断元素的equals方法，true——不存，false——存储
解决哈希冲突的方式：链地址法，建立公共溢出区，开放定址法
保证唯一性的方法：
覆盖Object中的hashCode和equals方法
hashCode是为了根据元素自身的特点确定哈希值
equals是为了解决哈希冲突
LinkedHashSet:取出有序
–TreeSet:二叉树数据结构，对元素进行排序，，不同步
排序方式：需要元素具备比较功能，所以元素需要实现Comparable接口，
覆盖comparreTo方法。
如何保证元素唯一性？
比较方法的返回值是否为0.
需求中也有这样一种情况，元素具备的功能不是所需要的，也就是说不想按照自然排序的方法，而是按照自定义的排序方式，对元素进行排序，而且存储到TreeSet中的元素万一没有比较功能
应该如何排序呢？这时需要使用——让集合具备比较功能。
实现Comparato接口，覆盖compare方法，2将comparator接口的对象作为参数传递给TreeSet的集合参数。
技巧：
jdk1.2以后出现的集合框架中的常用子类对象，存在的规律
前缀名是数据结构名，后缀名是所属体系名
ArrayList:数组结构，查询快，list，可以重复，可以增删改查
LinkedList:链表结构，增删快，
HashSet:哈希表，hashcode和equals查询速度更快，不保证有序，不可以重复
LinkedHashSet:链表+哈希表
TreeSet：二叉树，可以排序。就要想到2种排序方式：一种是自然排序Comparable，一种是比较器Comparator
